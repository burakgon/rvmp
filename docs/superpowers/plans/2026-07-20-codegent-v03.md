# codegent v0.3 Part 1 — Universal Terminal-State Tier Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **v0.3 is written as a just-in-time SEQUENCE (spec §16), one plan per independent subsystem.** This is Part 1 of 4:
> - **Part 1 (this plan): Universal terminal-state tier** — daemon-only, no network; the §6.1 payoff; closes v0.2's biggest debt (silent-wedge) and adds the mark-state override + §9.2 watchdog.
> - Part 2: Relay + pairing + Web Push (network; libsodium X25519 + secretstream; `docs/research/sshx-relay-e2e.md` verdict binds).
> - Part 3: Review queue + diff view + stale cascade + Send-back + PR tracking (closes Review→diff / Done→read-only-diff click targets).
> - Part 4: Installer + `npx codegent-cli` + service + first-run + Settings (+ CI provisioning, package layout for the compiled sidecar).
> Parts 2-4 are written just-in-time, folding in Part 1's learnings.

**Goal:** codegent orchestrates ANY recognizable agent — not just Claude Code and Codex — by classifying a TUI-resident agent's state (working / needs-input / idle-done) from OUTSIDE, content-free, per the §6.1 doctrine. This makes the universal tier real (spec §1/§6), and it structurally closes the v0.2 silent-wedge (a stopped-without-completing agent now becomes a visible `silent` attention flag instead of a card wedged in working.running forever).

**Architecture:** A daemon-side detection subsystem (`apps/daemon/src/detect/`) with a layered, content-free classifier — process-tree identity → OSC title/progress → manifest-scoped screen patterns → quiescence-as-readiness-gate-only — feeding a new `UniversalAdapter` (implements the existing `AgentAdapter` from v0.2) that emits the same `AdapterSignal`s the engine already consumes. Detection runs off the daemon's server-side terminal grid (the ring/PTY already parse bytes); only state ENUMs leave the daemon. Mechanisms + thresholds are adopted verbatim from `docs/research/herdr-agent-state.md` (`b580103`) and `orca-agent-state.md` (`80e6322`) — never re-tuned.

**Tech Stack:** Bun ≥1.3.14 + TS 7 monorepo (existing packages/protocol, apps/daemon, apps/web). Agent manifests in TOML (a tiny TOML reader — Bun has `Bun.TOML.parse` built-in, already used by the Codex adapter). No new heavyweight deps. Relay/review/install come in Parts 2-4.

### Task 4: Manifest engine (Layer 3)

**Files:**
- Create: `apps/daemon/src/detect/manifest.ts`
- Test: `apps/daemon/test/detect/manifest.test.ts`

**Interfaces:**
- Consumes: the Task-1 `screenGrid` contract (`{rows, oscTitle, oscProgress}`); `Bun.TOML.parse`.
- Produces: `loadManifest(toml: string) → Manifest` and `evaluate(manifest, grid) → {state: "idle"|"working"|"blocked"|"unknown", ruleId: string, fallback: boolean}`. A manifest is a TOML list of rules `{id, state, priority: i32, region, contains[] (lowercased substring), regex[], line_regex[], all[]/any[]/not[] gates, skip_state_update}` — **highest-priority matching rule wins** (herdr `manifest.rs:423-446`); complexity caps ≤128 rules / ≤512 gates / ≤1024 matchers / ≤512 chars-per-matcher (herdr `:264-269`). Regions: `whole_recent`, `bottom_lines(N)`, `bottom_non_empty_lines(N)`, `prompt_box_body` (between the last two horizontal `─` rules), `after_last_horizontal_rule`, `osc_title`, `osc_progress` (herdr `manifest.rs:1072-1095`). **Strict-positive fallback: a known agent with no matching rule ⇒ `idle`, tagged `default_known_agent_idle_fallback` — NEVER blocked** (herdr `:14,526-541`). `skip_state_update` rules mark agent-owned viewer screens (CC ctrl+o transcript) → detection freezes.

- [ ] **Step 1: failing tests** — parse a small manifest; highest-priority-wins (two matching rules, higher priority chosen); each region extracts the right slice from a grid fixture; `contains`/`regex`/`line_regex` + nested `all/any/not` gates; complexity caps reject an over-large manifest; unknown screen → idle-fallback (never blocked); `skip_state_update` freezes; a malformed manifest fails to load with a clear (non-crashing) error.
